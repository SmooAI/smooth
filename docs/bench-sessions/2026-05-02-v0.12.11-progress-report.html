  • Heartbeat — ac2f660 — Daemon-side dispatch_ws_task_sandboxed spawns a heartbeat task that posts [PROGRESS] sandbox running (Ns elapsed, heartbeat #N) every 30 s during exec. exec_in_sandbox is blocking, so without this the pearl's comment count was flat for the whole sandbox lifetime.
  • Store-resolution — dd64d79 — Bench's locate_pearl_store_dir() now prefers ~/.smooth/dolt/ (where the daemon writes since launchd starts it from $HOME) over the repo-walked .smooth/dolt. Heartbeat comments and bench polls were landing in different stores; bench never saw a single comment.
  • IDLE-on-exit — e0b2dd8 — Dispatch posts a terminal [IDLE] on exec_in_sandbox error and on non-zero runner exit, and reverts the pearl status to Open. Drops worst-case task-failure latency from ~10 minutes (full grace) to immediate. This is the fix that made take-7 task 2 (a genuine FAIL) terminate at 708 s instead of 1283 s.
  • Cost reporting — 77e49e3 — dispatch_ws_task_sandboxed posts [METRICS] cost_usd=X iterations=Y to the pearl right before closing on the success path. chat_driver::extract_cost() scans pearl comments for the latest [METRICS] line and returns the parsed value. Both bench early-return paths now report real cost instead of hardcoded 0.0. Take 7 (in flight before this fix landed) still reports $0.00; the next bench run carries actual spend.

Cost reporting (new)

Up through this report's "stale" cost column, every bench score JSON has carried cost_usd: 0.00 on every task because the chat-agent dispatch path returned HeadlessOutput.cost = 0.0 hardcoded — the dispatched teammate's actual LLM spend was tracked inside the daemon's final_cost_usd aggregator but never crossed back to the bench harness.

The fix uses pearl comments as the bridge. When the operator-runner finishes successfully, Big Smooth's dispatch_ws_task_sandboxed writes a single [METRICS] cost_usd=<f64> iterations=<u32> comment to the pearl immediately before closing it. The comment is part of the pearl's history, so any poller can read the dispatch's actual spend without subscribing to the WebSocket event stream.

Bench-side, chat_driver::extract_cost(&[PearlComment]) scans for the latest [METRICS] line, parses cost_usd=X, returns it. Falls back to 0.0 for any pearl that didn't get a Completed event (errored runs, deadline timeouts, pre-fix runs).

What this means for take 7: the run is still going under v0.12.11 e0b2dd8 (one commit before the cost-wiring fix), so the score JSON it produces will still say cost_usd: 0.00. The next bench run, after restarting the daemon on 77e49e3 or later, will carry real numbers. Worth a fresh run for the take-8 baseline.
