Round-2-sandbox-rust failure (the only ❌)

Real model wobble + missed iteration. Fixer (qwen3-coder-flash) wrote a HashMap::insert with a &char where char was needed. The Rust compiler emitted the diagnostic and the fix hint:

error[E0308]: mismatched types
  --> src/lib.rs:85:24
   | mapping.insert(next_letter, digit);
   |                ^^^^^^^^^^^ expected `char`, found `&char`
help: consider dereferencing the borrow
   | mapping.insert(*next_letter, digit);
   |                +

The dispatch ran ONE compile attempt and gave up. The fix is a one-character edit (*) that the compiler literally suggested — a working iteration loop should have caught this on turn 2. Two fixes worth filing as follow-up pearls:

  1. Operator iteration loop should retry on rustc compile error — currently exits after a single E-class error, even when rustc emits a one-line help: suggestion. Should at minimum retry once with the diagnostic + suggestion fed to the fixer role.
  2. Fixer prompt hardening on Rust borrow ergonomics — qwen3-coder-flash wobbles on HashMap::insert ownership semantics. A few-shot example in the fixer role's prompt would help, OR routing rust tasks through smooth-coding-kimi (thinking model) as a fallback.

What this run proves

  • Multi-role default routing solves real bench tasks with no env-var overrides. SMOODEV-800 lands the bench-winning config as the new baseline.
  • Cost flows end-to-end through the streaming path. Two-layer fix landed: agent's local ModelPricing fallback (since LiteLLM streaming reports Some(0.0)), plus [CHAT_METRICS] aggregation that surfaces orchestrator spend the runner-side [METRICS] alone misses.
  • Direct mode is ~7× faster for python and ~1.7-2.4× faster for rust. Sandbox tax is real; pick direct for fast iteration when sandbox isolation isn't required.
  • Round-to-round variance is low for direct python (42s → 35s). Sandbox rust has the widest variance (496s pass → 251s fail). Compile-error iteration loop is the gap.
