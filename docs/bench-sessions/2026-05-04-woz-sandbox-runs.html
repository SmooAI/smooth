Follow-up: full A/B/C/D matrix

The intended matrix — sandbox vs direct-mode × Gemini-direct vs smooth-* aliases — ran into multi-cell daemon-restart fragility:

  • Direct mode requires a host-triple operator-runner binary. Built and codesigned at target/release/smooth-operator-runner during the matrix attempt. ✓
  • Direct-mode chat dispatches were returning empty data. Probable cause: bench's multi-step dispatch prompt (pearls_create + teammate_spawn + return ONLY pearl id) is unreliable on Gemini Flash Lite for tool calls. Fixed by pinning Big Smooth's chat-agent to smooth-reasoning-kimi for the dispatch turn (Kimi K2-Thinking has proven tool-use compliance).
  • Daemon-restart-between-cells. launchctl setenv/unsetenv + kickstart -k was unreliable: env-var changes weren't always picked up by the new daemon process; sometimes the daemon stayed in direct mode after unsetenv. Need a more robust toggle (e.g. exec the daemon directly with a runtime flag instead of relying on env-var inheritance through launchd).

Filed as follow-up. The matrix will be valuable once those bench-harness/orchestration issues are resolved; the data above proves the Woz architecture itself works.

What this DOES prove

  • End-user simulation works. Three real-solves under the architecture where the bench process makes zero direct dolt reads or writes — everything goes through /api/chat and /api/pearls/{id}.
  • Reproducibility. Three independent solves on the same task, under sandbox dispatch, all green. Wall times 16.8–20.2 min, consistent with the operator's actual work (cold-start sandbox + tool installs + agent loop).
  • Empty-reply guard fires correctly. Big Smooth's chat-agent occasionally returns data: ""; the supervisor now treats that as Failed and rolls back the synthetic user turn. Confabulation drift eliminated.
  • 30s tick → 3× more coaching turns than the original 60s default, without runaway latency. The 12-iteration cap on the chat-agent catches loops before they hit the 900s ceiling.
  • New file tools for Big Smooth. read_file, glob_files, grep_files shipped this session — Big Smooth can now compose Claude-Code-style status replies (diffs, tool calls, test output) by reading the workspace directly instead of round-tripping through teammate prompts.

What this DOESN'T prove (yet)

  • Direct-mode (no-sandbox) parity — needs the matrix follow-up.
  • Smooth-* alias parity for the supervisor LLM — needs the matrix.
  • Real cost per task — LiteLLM pricing fix is deployed to main but the production rollout was in flight at the time of writing.
  • Behavior on harder tasks (rust/alphametics, java/alphametics) where take 9 saw regressions — will measure once the matrix infrastructure stabilises.
